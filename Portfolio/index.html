PMS (Project Management System)

PMS is a Kanboard-based project and workflow management system that we extended for a hiring/interview workflow. It supports Kanban boards for tickets/tasks, structured candidate/job tracking, interview scheduling per ticket, centralized interview artifacts (OneDrive/Drive), and automated notifications. I worked on custom plugins and theme-level enhancements including an interview/events module, user-specific notification rules, Microsoft Teams/Slack notifications, performance improvements for board drag-and-drop and lazy-loaded task details, and a custom API layer to create/update/sync tickets with external systems while logging requests and handling rich HTML content with embedded images.

Key Contributions
  • Customized Kanboard into an internal PMS with hiring/interview workflow features.
  • Built an interview/events module per ticket with filters (date range, call/round position, technology multi-select) and project-level views.
  • Implemented automated notifications (email + Teams/Slack) including interview reminders and assignee targeting.
  • Added OneDrive/Google Drive integrations for interview artifacts, plus video-to-audio conversion via FFmpeg and upload verification.
  • Integrated Google Gemini to generate description summaries, transcript summaries, and a final consolidated interview-feedback summary.
  • Extended the JSON-RPC API for task creation/update/sync with request logging, pagination support, and base64 image extraction/storage.

Technology Stack: PHP (Kanboard), Composer, MySQL, Kanboard plugins/themes, JSON-RPC API, OAuth (Google/Microsoft), Microsoft Teams/Slack webhooks, SMTP email, OneDrive + Google Drive APIs, FFmpeg, Google Gemini API, JavaScript, CSS
